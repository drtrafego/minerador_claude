    Chave privada SSH (cat ~/.ssh/id_rsa)

Arquitetura
Como o sistema roda em producao

Frontend no Vercel (serverless), backend no seu servidor (containers Docker).

Fluxo de deploy

Git push (master)

Voce faz push. GitHub detecta a mudanca.

→

Vercel (frontend)

Deploy automatico do Next.js em ~30s.

+

GitHub Actions

SSH no servidor, git pull, docker compose up --build.

Em producao

Vercel Edge

Next.js App Router, Server Actions, API routes. Escala automaticamente.

↔

Neon PostgreSQL

Banco de dados serverless. Lido por Vercel e pelo worker.

↔

Servidor (Docker)

Worker processa jobs da fila pg-boss. Scrapling faz scraping.

Primeiros passos
Subindo o servidor pela primeira vez

Execute uma unica vez no servidor. Depois e tudo automatico via GitHub Actions.

1

Instalar Docker

Caso nao tenha Docker instalado no servidor.

curl -fsSL https://get.docker.com | sh sudo usermod -aG docker $USER && newgrp docker
2

Clonar repositorio

Clone o projeto em /opt/minerador.

git clone https://github.com/DR-TRAFEGO/minerador_claude.git /opt/minerador cd /opt/minerador
3

Criar arquivo de variaveis

Crie o .env.prod com as variaveis do servidor (listadas acima).

nano /opt/minerador/.env.prod # Cole as variaveis e salve (Ctrl+X, Y, Enter)
4

Subir os containers

Builda as imagens e sobe worker + scrapling em background.

docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build
5

Verificar logs

Confirma que o worker esta conectado ao banco e aguardando jobs.

docker logs minerador_worker --tail 30 -f
6

Configurar GitHub Secrets para deploy automatico

Adicione SERVER_HOST, SERVER_USER e SERVER_SSH_KEY no GitHub. A partir dai, todo push no master atualiza o servidor sozinho.

cat ~/.ssh/id_rsa # copie e cole como SERVER_SSH_KEY no GitHub
Fluxo principal
Como um lead vira cliente

O sistema funciona em 5 etapas automaticas, com intervencao humana so quando necessario.

1

Mineracao de Leads

O sistema raspa fontes como Google Maps, Instagram, LinkedIn e listas CSV. Cada lead coletado vira um registro com nome, contato, empresa e dados brutos.

Scrapling (Python) Google Maps Instagram LinkedIn CSV / Webhook
2

Qualificacao com IA

Claude analisa cada lead contra o ICP (perfil de cliente ideal) da campanha. Classifica como qualificado, nao qualificado ou pendente e atribui uma nota de 0 a 100.

Claude Sonnet ICP configuravel Nota 0-100 Motivo em texto
3

Outreach Automatizado

Para cada lead qualificado, o sistema abre um thread no canal configurado e dispara a mensagem inicial. Follow-ups sao enviados automaticamente com os intervalos configurados na campanha.

WhatsApp Meta Cloud API WhatsApp UazAPI Instagram DM Email (Gmail)
4

Agente de Resposta

Quando o lead responde, o agente IA assume a conversa. Responde perguntas, supera objecoes e tenta marcar uma reuniao. Se detectar palavras-chave de handoff, transfere para humano.

Resposta automatica Tom configuravel Max respostas auto Handoff humano
5

Pipeline e Fechamento

Leads que avancam ficam no Kanban do pipeline. Arraste entre colunas (contato, reuniao marcada, proposta, ganho, perdido) e registre atividades manualmente ou via automacao.

Kanban drag-and-drop Estagios customizaveis Historico de atividades
